分析卡池数据函数：

```js
//分析单个卡池中的数据
function analyzePool(records) {
    //const theTotal = records.data.length;
    const theTotal = records.length;//设定参数为纯数组，优化接口结构
    let fiveStarCount = 0;
    let fourStarCount = 0;

    let pity = 0;//当前总抽数
    let pityList = [];//记录每次出金的抽数

    for (const record of records) {
        pity++;
        if (record.qualityLevel === 5) {
            fiveStarCount++;
            pityList.push(pity);
            pity = 0; // 重置出金抽数，新一轮保底开始
        }
        else if (record.qualityLevel === 4) {
            fourStarCount++;
        }
    }
    const avgPity = pityList.length ? (pityList.reduce((a, b) => a + b, 0) / pityList.length) : 0;

    return {
        total: theTotal,
        fiveStar: fiveStarCount,
        fourStar: fourStarCount,
        avgPity: avgPity,
        pityList
    };
}

module.exports = {
    analyzePool
};
```

## 五星名称

接下来需要记录每个五星的名称。根据`resourceId`来标注五星名称。  
将pityList数组扩展，加入`name`属性。  
预想根据`resourceId`属性来逐个匹配不同元素的名称，实际实现仅需直接加入`name`属性。  

```js
pityList.push({
                count: pity,
                name: record.name
            });
```

当前实现将发生错位，name总是后移一位。  

### 修改思路

数组从新向旧push。  
旧  
|9抽--b  
|11抽--c  
|32抽  
新  
for循环  
|32抽--c  
|11抽--b  
实际匹配的name是上一条五星记录的name。  

传入的record顺序为从新到旧。需要将record按时间顺序从旧到新排序：

```js
records.sort((a, b) => new Date(a.time) - new Date(b.time));
```

方案不可行。由于十连记录的time属性相同，无法实现正确的排序。  

直接倒序排列：

```js
records = records.reverse();
```

结果完全匹配。
