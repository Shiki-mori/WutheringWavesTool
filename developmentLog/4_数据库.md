## 环境配置

### 安装mysql

```bash
npm install mysql2
```

此方法在openSUSE不适用。需要：

```bash
sudo zypper install mysql mysql-client
```

查找失败后会推荐安装`mariadb`。这是`mysql`的完全兼容替代品，常用于openSUSE系统。  
`mariadb`是`mysql`的开源分支。  
待安装的几个包中，`mariadb`是数据库服务，`mariadb-client`是mysql命令行工具。  
可以选择mariadb，也可以去官方网站获取仓库rpm包：`https://dev.mysql.com/downloads/repo/zypper/`。  
选择suse15版本（对于tumbleweed通常如此）。下载。  
运行：  

```bash
sudo zypper install https://repo.mysql.com//mysql84-community-release-sl15-1.noarch.rpm
```

执行后会显示签名校验失败警告，选择忽略即可。

刷新仓库并安装：

```bash
sudo zypper refresh
sudo zypper install mysql-community-server
```

选择信任密钥即可。
在后续的冲突中均选择忽略。忽略后，最终检查mysql版本，发现被mariadb覆盖。  

彻底移除mariadb相关包：

```bash
sudo zypper remove -u mariadb-client mariadb-common
```

重新运行mysql的安装流程：

```bash
sudo zypper clean -a
sudo zypper install --force mysql-community-server
```

使用force参数是为了确保之前被跳过的文件被正确覆盖安装。  
验证二进制文件路径，检查`mysql`命令指向：  

```bash
which mysql
mysql --version
```

本次尝试依旧失败。再次尝试安装。  
卸载mysql所有包：  

```bash
sudo zypper remove mysql-community-server mysql-community-client mysql-community-client-plugins mysql-community-libs mysql-community-common
```

重新安装：

```bash
sudo zypper refresh
sudo zypper install mysql-community-server
```

仍然安装失败，但是知道了原因：  
error: Key b7b3b788a8d3785c (MySQL Release Engineering <mysql-build@oss.oracle.com>) expired on 2025-10-23 01:26:50  
官方仓库旧的GPG签名密钥过期了，因此系统拒绝安装。  

需要手动导入Mysql发布的新版密钥。  

下载并信任mysql的新密钥：

```bash
sudo rpm --import https://repo.mysql.com/RPM-GPG-KEY-mysql-2026
```

刷新仓库，确保zypper认到新的密钥：

```bash
sudo zypper refresh
```

重复安装流程。
失败了。

解决了！按照以下步骤安装：  
直接调用底层的rpm工具，以更灵活的方式忽略签名校验。  
先下载所有的包（不安装）：

```bash
sudo zypper install --download-only mysql-community-server
```

进入缓存目录执行强制安装：

```bash
cd /var/cache/zypp/packages/mysql-*
```

使用tab自动填充即可找到路径，或者每到一层路径使用ls命令查看目录，再进入。  
在该路径下使用ls列出所有.rpm文件，逐一强制安装：  

```bash
sudo rpm -ivh --nosignature --nodeps mysql-*.rpm
```

使用`--nosignature`忽略过期的密钥。

安装完成后验证：

```bash
mysql --version
```

得到：

```
mysql  Ver 8.4.8 for Linux on x86_64 (MySQL Community Server - GPL)
```

### 初始化mysql

由于通过rpm强制安装，可能没有触发自动初始化。手动触发：  

```bash
sudo mysqld --initialize --user=phrolova
```

启动服务：

```bash
sudo systemctl start mysql
```

查看初始密码：

```bash
sudo grep 'temporary password' /var/log/mysql/mysqld.log
```

得到的temporary密码：`gc!BlwZyK9Eg`

修改密码：

```bash
mysql_secure_installation
```

显示 `Error: Access denied for user 'root'@'localhost' (using password: YES)`。  
失败了，直接使用跳过权限表的方式来重置root密码。  

停止Mysql服务：

```bash
sudo systemctl stop mysql
sudo killall -9 mysqld
```

绕过登录验证，直接进入数据库后台：

```bash
sudo mysqld --user=mysql --skip-grant-tables --skip-networking &
```

进入客户端：

```bash
mysql -u root
```

执行：

```sql
FLUSH PRIVILEGES;
-- 注意：MySQL 8.4 要求密码必须足够复杂（大小写+数字+特殊字符）
ALTER USER 'root'@'localhost' IDENTIFIED BY 'MyNewPassword@123';
FLUSH PRIVILEGES;
EXIT;
```

清除该不安全的进程，重新以正常方式启动服务：

```bash
sudo killall -9 mysqld
sudo systemctl start mysql
```

使用新密码登录：

```bash
mysql -u root -p
```

输入设置的密码。

设置开机自启动：

```bash
sudo systemctl enable mysql
```

初始化：

```bash
sudo mysql_secure_installation
```

将会提示设置root密码；是否删除匿名用户anonymous users；是否禁止root远程登录；是否删除测试数据库。后三个均选择yes。

### 创建项目数据库

登录mysql：

```bash
mysql -u root -p
```

输入密码，进入`mysql>`。

创建数据库：

```sql
CREATE DATABASE analyzer;
```

查看：

```sql
SHOW DATABASES;
```

一定要带分号。可以使用ctrl+C取消当前输入。

### 创建专用用户

创建：

```sql
CREATE USER 'analyzer_user'@'localhost' IDENTIFIED BY 'Analyzer@123';
```

授权：

```sql
GRANT ALL PRIVILEGES ON analyzer.* TO 'analyzer_user'@'localhost';
```

刷新权限：

```sql
FLUSH PRIVILEGES;
```

### 建表

切换数据库：

```sql
USE analyzer;
```

建表：

```sql
create table analyzer_records(
    id bigint auto_increment primary key,
    uid varchar(50),
    resource_id varchar(30),
    resource_name varchar(100),
    is_up boolean,
    count int, --总抽数
    quality_level int,
    pool_type int,
    time datetime,
    created_at datetime default current_timestamp,
    unique key uniq_record(uid,resource_id,time)
);
```

sql是强类型语言，每个字段必须定义完整类型。  
大小写不敏感，常用下划线为变量命名。  
varchar必须指定长度。  

查看：

```sql
show tables;
```

看到analyzer_records说明创建成功。  

## 附录-更改表结构

核心命令：`alter table table_name`。在进行以下所有更改前都需要输入此命令，且末尾不用分号

删除字段（drop column）：`drop column column_name;`

修改字段：  

- 修改字段名称（change）：`change source_name target_name source_type;`
- 修改字段类型（modify）：`modify column_name target_type;`

添加字段（add column）:`add column column_name column_type (after last_column_name);`  
可选择指定位置

修改唯一索引  
本表中唯一索引为：`unique key uniq_record(uid,resource_id,time)`  
删除索引：`drop index uqiq_record;`  
新增索引：`add unique key uniq_record(uid,resource_id,time);`

## 关于数据库应该存储原始数据还是分析后的结果--数据库的职责划分

数据库应该存储可复用的原始事实数据。  
数据库=数据源  

抓取数据--后端分析--存数据库（错误）  
抓取数据--数据清洗--存数据库--查询数据库--后端分析--返回前端（正确）

如果分析结果存入数据库，其值不可复用，不可验证，不可扩展。

数据库除了原始数据，可以存一部分派生数据。
